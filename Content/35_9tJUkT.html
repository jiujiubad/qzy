所属章节：3-初探 Rails

本章预计学习时间: 4小时

再学习13节就可以完成本章了

共 487 位第二期同学学习了本节

3-1 开始吧！

目标

建立你的 New Application

来，开始吧！在这一步骤的最后，我们会得到一个全新的（空白）Rails 应用程序。

步骤


步骤 1

cd 的意思是切换目录（Change Directory）

在终端机打这些字 ( Mac 或 Linux )

cd ~
cd ~ 切换（当前位置）到home目录


步骤 2

在终端机打这些字：

mkdir railsbridge

这会新增一个资料夹，让我们把项目放在里面。


步骤 3

在终端机打这些字：

cd railsbridge


步骤 4

确定一下你有没有之前练习用的代码在里面。

在终端机打这些字：

ls

这指令会列出 railsbridge 资料夹里面的文件。如果列出了旧的 suggestotron 应用程序，你可以删掉它们来避免干扰：

在终端机打这些字：

rm -rf suggestotron


步骤 5

在终端机打这些字：

rails new suggestotron

rails new 会用你给的名字来建立新的 Rails 项目

这次我们建一个新的项目叫 suggestotron 。我们马上就会详述它到底建立了什么。


步骤 6

在终端机打这些字：

cd suggestotron

cd 指的是切换目录 (change directory)。

cd suggestotron 切换到 suggestotron 做为成当前资料夹。


步骤 7

在终端机打这些字：

ls

ls 指的是列出东西 (list stuff)。

它会把目前资料夹的内容列出来给你看。


步骤 8

运行 Atom 编辑器，用项目的方式打开 suggestotron 资料夹。

选择 File > Add Project Folder...。

在右上角输入suggestotron，电脑会找到这给资料夹，点击选择它，如果一切没问题的话，Atom 会在视窗左边以树状结构列出资料夹的内容：

你会发现 rails new 开了一大堆资料夹和文件。今天我们要看的是这些：

（文件/资料夹 ：用途 ）

app/： 包含你应用程序的 controllers、models、views。你要改的东西大多是这些。

config/： 设定应用程序的执行阶段规则、路由设定（routes）、资料库等等。

db/：显示你目前资料库的 schema（结构定义），以及资料库的 migrations。

public/：这是唯一一个资料夹会是放什么就出现什么的。如果你把文件放里面，server 会直接响应，不会经过 Rails 的处理。

app/assets/：你会要把图片、JavaScript、stylesheets (CSS) 还有其他静态文件放在里面。现代的 Rails 应用程序使用一种叫做 Assets Pipeline 的东西，把在这资料夹里面的所有 JavaScript 和 CSS 档合并成一个文件来加速。

rails new 还建立了其他很多东西。大概可以写一本书来讲，所以我们现在先无视它们。
