silvia 提交了作业 4 天前
作业 - 作业三
1.   在網址列輸入網址，或是點網頁上的超連結
2.   瀏覽器解析這個 URL 找出 protocol、host、port 和 path
3.   如果是 HTTP，則組出 HTTP request 數據包
4.   查詢 DNS 用 host 找出對應的 IP 位址，組出 IP 數據包
5.   組出 TCP 封包並打開一個 TCP connection 到上述 IP 位址和 Port
6.   提交 HTTP Request 數據包
7.   Web 服務器(例如 Nginx)接收到 HTTP Request 封包後，解析其中的內容(特別是其中的 path)來決定如何回應：
¡   如果是靜態內容，也就是 HTTP Response 不會根據不同用戶、不  
    同時間等任何因素而改變，例如圖片、影片、CSS、JavaScript 
    等等。那麼 Web 服務器會直接將該檔案回傳給瀏覽器。
¡   如果是動態內容，也就是會根據不同登录的使用者、時間等參數來
    回傳不同內容，那個 Nginx 會將 HTTP Request 交給應用程序
    處理，例如 PHP、Ruby、Python、Node.js、Java 等程序，由該
    程序動態地根據不同參數和條件，搭配數據庫來撈取資料，然後
    組合出 HTML 網頁包成 HTTP Response 經由 Nginx 回傳給瀏覽器。
8.   瀏覽器接收到這個 HTTP Response，開始解析這份 HTML 網頁成為 Document Object Model (DOM) 一種樹狀的資料結構來對應 HTML 的標籤節點。
9.   瀏覽器會依序檢查 HTML 原始碼中，需要下載的資源網址 URL，例如 CSS、圖片、JavaScript 等等，就會再發 HTTP Request 請求下載回來 (依照上述步驟二到九)
10.  如果下載到 CSS 樣式表，就會去裝飾對應的 DOM 節點
11.  如果下載到 JavaScript 程序碼，則會執行它。瀏覽器上的 JavaScript 程序可以操作 DOM 節點，通常會用 jQuery 函式庫來做，來達到一些網頁動態特效。
12.  直到所有資源都下載完畢，瀏覽器執行完 CSS 和 JavaScript，才算大功告成完成(Page Load)。如果你又點擊網頁上的超連結，則又回到步驟一。

输入网址时到底发生了什么？
1、信是由信纸和信封构成，信纸写着要传输的内容，信封写着地址。
2、网络数据的传输类似于这个过程，网络数据是相当于信纸，会由里到外套上4个信封，每个信封有着不同的功能：
第一个信封（应用层）：规定着数据传输的格式，是网页、邮件、文件。
第二个信封（传输层）：电脑上运行着很多程序，这个信封上写着邮递员要将这个数据交给哪个程序，同时保证数据的可靠性。
第三个信封（网络层）：信件到达目的地需要经过很多中转站，犹如快件需要经过很多中转站才能到达目的地，该信封上规定了需要经过哪些中转站。
第四个信封（链路层）：这个信封写着终点的地址。
3、输入网址的过程：
首先：浏览器解析网址，如果是http协议，那么就将网页整理成header和payload的形式。
其次：遵照tcp协议，将数据进行拆分，并标上序号。
再次：IP协议，需要知道终点的ip，那么就需要DNS服务，能够进行域名解析。
最后：以太网协议，在信封上填写自己的Mac地址和接收方的Mac地址（刚开始不知道）

1、浏览器解析URL，什么应用协议、HOST、PORT和PATH 
2、如果是HTTPIE协议，则打包成HTTP数据包 
3、利用DNS协议，通过HOST找到对应的IP地址 
4、利用TCP协议，将数据包标序后打包成多个小数据包，然后发送到指定IP地址和PORT 
5、WEB服务器接收到数据包后，进行分析（特别是path）然后给出需要的回应给浏览器 
6、得到回应后的浏览器解析URL的CSS、JAVASCRIPT和图片等等，然后依次发出请求，重新执行上述步骤 
7、直到浏览器执行完所有的CSS和JAVASCRIPT，则页面成功打开

hubodashi 提交了作业 2 个月前
作业 - 作业三
浏览器的中输入网址后，网络数据协议分为四层，进过层层处理，大致有一下几个步骤：
1、输入网址后，浏览器解析这个网址。
2、对于HTTP，处理出request数据包；查询DNS，用host找出对应的IP地址，处里出IP数据包；处理出TCP封包并打开一个TCP connection到上述IP位置和POST。然后，提交HTTP Request数据包。
3、Web服务器接收到HTTP Request后，解析其中内容。如果是静态内容，Web服务器会直接将该文件回给浏览器。如果是动态内容，则会交给不同的应用程序处理，再由该程序捞取资料，然后组合出HTTP Response由服务器传给浏览器。
4、浏览器接收到这个HTTP Response后对其解析为对应数据。依序检查HTML原始码中需要下载的资源，然后再次发出HTTP Request请求下载，并进行对应处理。
5、所有资源下载处理完毕后，浏览器执行完CSS和JavaScript，就完成了一次Page Load。
